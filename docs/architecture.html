📱 HitHitKit iOS SDK

repo: github.com/DevVenusK/iOS-HitHit

호스트 iOS 앱의 탭·스크롤 상호작용을 디바이스 무관한 정규화 좌표로 수집해 서버로 직접 전송하는 1st-party 분석 SDK. 히트맵 렌더링은 범위 밖 — 수집만 한다.

Swift 5.9+iOS 14+SwiftPM 서드파티 의존성 0동의 기본 OFF좌표만 수집터치당 <0.5ms

수집 파이프라인

컴포넌트역할
TrackingWindow전역 탭 인터셉트. .ended에서 이동거리 ≤ tapSlop(10pt)일 때만 탭으로 기록(스크롤/드래그 제외). 좌표는 window 화면 기준으로 정규화.
ScrollTracker터치가 시작된 스크롤뷰를 자동 등록(스위즐 아님). CADisplayLink 10Hz로 scrollDepth(adjustedContentInset 반영) 샘플. 백그라운드 시 일시정지.
HitHitCollector공개 파사드: start·stop·setConsent·setScreen·track·flush·purgePendingEvents. 내부 상태는 lock으로 보호.
EventPipeline순수 게이트(CollectionGate)로 실행·동의·제외화면·샘플링 판정 → 버퍼 적재 → 전송(코얼레싱 드레인).
EventStoreJSONL 임시 버퍼. 전송 성공분만 라인 단위로 제거(중복/유실 방지), 실패분 보존.
DefaultHTTPUploaderendpoint로 JSON 배열 POST. 지수 백오프 재시도, 실패 시 로컬 보존.
HitHitCore순수 로직: HitHitEvent 스키마 · Normalization · TouchClassifier · HitHitError.

전송되는 이벤트 (wire format)

{ "schemaVersion":1, "id":"UUID", "type":"tap|scroll",
  "screen":"loan_detail",
  "x":0.42, "y":0.73,              // tap: 정규화 0~1
  "scrollDepth":0.65,              // scroll: 정규화 깊이 0~1
  "screenW":402, "screenH":874,    // 정규화 기준 화면 크기
  "device":"iPhone15,3", "orientation":"portrait",
  "ts":1719800000000 }             // epoch ms

콘텐츠·입력값·요소식별자는 절대 담기지 않는다. 좌표·화면이름·시간만.

🗄️ hithitkit-server Python 바이너리

repo: hithitkit-server (동반 서버 · 단일 실행 바이너리)

HitHitKit이 보낸 이벤트를 적재(serve)하고, 명령 한 번으로 그 데이터에서 히트맵 PNG를 생성(hitmap)하는 동반 서버. 저장소는 URL 스킴으로 교체 가능.

Python · stdlib http.servernumpy · scipy · Pillow PyInstaller 단일 바이너리저장소 플러그블

CLI

# 수집 서버
hithitkit serve --bind http://0.0.0.0:8899 --storage file:///data/events.jsonl

# 히트맵 생성 (tap | scroll | both)
hithitkit hitmap --storage file:///data/events.jsonl --type both --points --open

HTTP 엔드포인트

메서드 · 경로동작
POST /이벤트 적재. JSON 배열 또는 단건 → storage.append → {"ok":true,"stored":N}. 파싱 실패 시 400.
GET /health헬스체크(ok).
GET /stats적재 통계: total · by_type · by_screen.

저장소 (URL 스킴)

file://…jsonlsqlite://…dbs3://bucket/prefix postgres://…http(s)://…/ingest (write-forward)

히트맵 렌더 파이프라인

flowchart LR A["storage.read_all()"] --> B["filter_events

같은 (screen, device, size, orientation) 버킷끼리만 집계해야 좌표가 의미를 가진다 (v0.0.5부터 SDK가 window 기준으로 정규화해 화면당 일관됨).

🔒 프라이버시 · 신뢰 원칙

  • 동의 마스터 스위치 — 기본 OFF. setConsent(true) 전엔 수집·전송 0건. 철회 시 신규 수집·업로드 중단 + purgePendingEvents().
  • 좌표만 — 텍스트·입력값·요소식별자 미수집. screen 이름엔 PII 금지(고정 심볼릭 이름).
  • 민감화면 제외 — excludedScreens로 로그인·계좌·금액 화면 차단.
  • 성능 예산 — 터치당 메인스레드 <0.5ms(정규화+enqueue만), 인코딩·IO·네트워크는 100% 백그라운드.
  • 서버 dedup — 이벤트 id(UUID)로 재시도·ACK 유실 시 서버 측 멱등 처리.
